Crail is a open source user-level I/O architecture for the Apache data processing ecosystem designed from ground up for high-performance storage and networking hardware (40-100Gbps RDMA, flash, GPUs, etc)



The backbone of the Crail I/O architecture is CrailFS, a high-performance distributed data store for temporary data in analytics workloads. CrailFS implements a multi-tiered file system interface across a cluster of RDMA interconnected storage resources such as DRAM, flash, GPU memory, etc. It incorporates several unique properties:
  • User-level I/O: CrailFS is built explictly for user-level I/O (RDMA, NVMef) allowing storage and networking hardware to directly access I/O memory within the data processing engine. Bypassing OS, JVM, etc. permits delivering bare-metal I/O performance to analytics workloads.
  • Byte-addressable: Storage resources (DRAM, flash) are treated as byte-addressable memories, which results in very efficient random access performance.
  • Storage-tiering: CrailFS exploits fast networks by implementing horizontal storage tiering. With horizontal tiering, higher performing storage resources are filled up across the entire cluster prior to using lower performing resources, resulting in a more effective usage of storage.
CrailFS can serve as a source/sink for input/output data of analytics workloads through its HDFS adaptor. CrailFS also serves as a high-performance data store for some of the performance critical data operations such as shuffle or broadcast data. Crail specific implementations of these operations are part of dedicated modules. As an example, Spark-IO implements a high-performance shuffle engine for Spark using Crail.
